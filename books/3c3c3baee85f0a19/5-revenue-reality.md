# 第5章 収益の現実——$600投資して¥5,000回収

## Day 0: $2

記録を始めた日の収益は$2。itch.ioでゲームが1本売れた。Azure Flame、ハクスラ系のローグライク。$2。

Claude Codeの月額は$200。3ヶ月で$600。$2では話にならない。

## Ops Kit: $79 → $19 → PWYW → 0件

最初の商品はOps Kit。Claude Codeの自律運用に使っているファイル群をパッケージにした。$79。

売れなかった。

$19に値下げした。売れなかった。

PWYW（Pay What You Want、自由価格）にした。$0から買える。タダでもいい。

0件。

94回のクリックを追跡した。94人が商品ページを見た。誰も買わなかった。タダでも。

:::details CC補足: PWYWモデル
Pay What You Wantは購入者が金額を自由に決めるモデル。Gumroadでは$0+（最低$0）に設定可能。Humble BundleやBandcampで実績があるが、知名度ゼロの商品では「$0でも要らない」が現実。
:::

原因は後になって分かった。商品ページに飛んでも「これが自分に必要だ」と思う前段階がない。知らない人が知らない商品を見ても、欲しいとは思わない。当たり前のことだが、当時は気づかなかった。

## 初めてのZenn Book売上: ¥800

3月11日。転機が来た。

Zennで有料Bookを公開していた。「[Claude Codeを本番品質にする——700+時間の自律稼働から生まれた実践ガイド](https://zenn.dev/yurukusa/books/6076c23b1cb18b)」。¥800。

売れた。1冊。

CVR（コンバージョン率）は10%。10回表示されて1回売れた。Gumroadの0%とは桁が違う。

なぜ売れたのか。Qiitaで無料の技術記事を公開していた。記事を読んだ人が「もっと詳しく知りたい」と思った。Zennに来た。Bookを見つけた。買った。

**無料で価値を体験した後に、有料商品が売れる。** Gumroadで0件だった理由もこれで説明がつく。いきなり商品ページに飛ばしても、信頼がないから買わない。

## 全プラットフォーム収益: 約¥5,000

3月31日時点の全プラットフォーム合計。

| プラットフォーム | 収益 | 備考 |
|---|---|---|
| itch.io | $12 | Azure Flame |
| PromptBase | $2.99 | プロンプト販売 |
| Zenn Book | ¥2,780 | 4件（手数料後） |
| Gumroad | $0 | 0件 |
| npm | $0 | DLは増えたが収益ゼロ |
| **合計** | **約¥5,000（≈$33）** | |

コスト$600に対して約¥5,000（≈$33）。回収率約5.5%。赤字▲約$567。

## npm DLの幻想

npmパッケージのダウンロード数が増えていた。cc-audit-logが週371ダウンロード。cc-health-checkが週367ダウンロード。

嬉しかった。ユーザーがいる。使ってくれている。

調べた。ボットだった。

npmのダウンロード数にはミラーリングbot、CI/CDパイプラインのインストール、セキュリティスキャナーが含まれる。実際にツールとして使っている人間のダウンロードがいくつあるか。証拠がなかった。

:::details CC補足: npmダウンロード数の内訳
npmの公開ダウンロード統計には以下が含まれる: (1) 実ユーザーのnpm install、(2) CI/CDパイプラインの自動インストール、(3) レジストリミラーの同期、(4) セキュリティスキャナーの巡回。小規模パッケージでは(2)-(4)が大半を占めることがある。正確な実ユーザー数を知る方法はnpm側には存在しない。
:::

npm作業を停止した。ダウンロード数は収益にゼロの影響。メトリクスが上がっても$0は$0。

## Gumroad完全死亡

3月28日。Gumroadの全商品のメトリクスを確認した。

全商品0 sales。$0 revenue。30日間で。

しかも、商品説明を修正しようとしたらできなかった。GumroadのUI（React + Formik）がCDP操作を受け付けない。テキストを入力しても、Reactの内部stateが更新されず、保存ボタンを押しても元の内容に戻る。

結局、自分が手動で2分で修正した。AIで45分試して失敗したことを、人間が2分で終わらせた。

## CVR 0.012%のボトルネック

3月29日。全チャネルのデータを集計した。

- npm: 21,706ダウンロード/週
- GitHub: 6,083クローン/14日
- Qiita: 34,248ビュー

膨大なトラフィック。購入は4件。CVR 0.012%。

99.988%の人が素通りしている。

原因は明確だった。「認知→関心→検討→購入」のファネルで、**「関心」のステップが存在しない。** 記事を読んでツールを知る。でもそこから「もっと深く知りたい」「自分にも必要だ」と思うステップがない。

唯一うまくいったパターンは、Qiita記事→Zenn Book。無料記事で「hookは役に立つ」と体験してもらい、「もっと知りたい」と思った人がBookを買う。

## $600の赤字を正直に公開した

3月15日。Twitterにこう書いた。

> $600 spent. $16.26 earned. 2.7%

赤字を隠す選択肢もあった。でもそれは自分が記事で書いている「誠実さ」と矛盾する。

正直に書いた。「これがリアルだ」と。

note記事でも収支報告を書いた。$600投資、¥5,000回収。赤字▲$567。隠さない。

信用は隠すことではなく晒すことで得る。少なくとも自分はそう信じている。

## それでも黒字化の道筋は見えた

¥5,000。赤字。でも道筋は見えた。

Zenn Book 1冊目は¥800で4件売れた。月1-2件ペース。小さいが確実に動いている。

Qiitaの記事は35,000ビューを超えた。読者がいる。価値を感じてくれている人がいる。

問題は「トラフィックを収益に変換する仕組み」がなかったことだ。今は無料記事→有料Bookの導線がある。

$200/月のコスト（約¥30,000）を回収するには、¥800のBookを月約45冊売る必要がある（Zenn手数料で1冊あたりの手取りは約¥675）。現状の10倍以上。簡単ではない。

ただし「簡単ではない」と「不可能」は違う。この本を書いている時点で、まだ赤字だ。

その事実から目を逸らさない。

## 🔧 実践ガイド: AI事業の収益設計

ここまでの失敗から得た教訓を、再現可能なフレームワークにまとめる。

### チャネル選択マトリクス

全てのプラットフォームに投稿するのは間違いだった。3ヶ月のデータから、チャネルの効率は以下のように差がついた:

| チャネル | トラフィック | 収益 | 効率 | 判定 |
|---------|------------|------|------|------|
| Qiita | 35,000v | → Zenn Book ¥2,780 | 最高 | ✅ 継続 |
| Twitter | — | → itch.io $12 | 低いが実績あり | ✅ 維持 |
| GitHub | 6,000 clones | → $0 | 認知のみ | ⚠️ 間接効果 |
| npm | 22,000 DL/週 | → $0 | ボット | ❌ 停止 |
| dev.to | 959v/30記事 | → $0 | Qiitaの1/30 | ❌ 停止 |
| Gumroad | 94 clicks | → $0 | 完全死亡 | ❌ 停止 |

**判断基準**: 同じ労力をかけた時、どのチャネルが最もリターンが大きいか。差が3倍以上あるなら、効率の悪い方をやめる（第8章のサンクコスト参照）。

### ファネル設計テンプレート

「いきなり売る」は失敗する。94クリック0購入がその証拠。

```
失敗パターン:
  Twitter投稿 →「商品作りました！」→ Gumroadリンク → 0購入

成功パターン:
  Qiita無料記事 →「hookって便利だな」→ 記事末尾のBook紹介
  → Zenn Bookページ →「もっと知りたい」→ 購入
```

**設計手順:**

1. **無料で価値を体験させる場所を作る**: Qiita記事、Zenn無料記事、GitHub README
2. **「もっと知りたい」と思わせる**: 無料コンテンツの末尾に「体系的に学べるBookがある」と自然に案内
3. **購入ページで「自分に必要だ」と確信させる**: 目次、無料公開章、読者の声
4. **購入後に「得した」と思わせる**: コンテンツの品質が全て。ここで手を抜くと信用が死ぬ

### 「売れない」を診断するチェックリスト

商品が売れない時、原因は以下のどこかにある:

- [ ] **認知**: そもそも商品の存在を知られていない → 無料コンテンツで露出を増やす
- [ ] **関心**: 知っているが興味がない → 無料体験で価値を感じさせる
- [ ] **信頼**: 興味はあるが信用していない → 実績、数字、正直な情報公開
- [ ] **価格**: 信用しているが高い → 価格テスト、PWYW、無料版提供
- [ ] **決済**: 買いたいが手段がない → 決済プラットフォームの見直し

94クリック0購入の場合、「認知」はクリアしている。問題は「関心」と「信頼」。これがファネルの「関心ステップ」が必要な理由。
