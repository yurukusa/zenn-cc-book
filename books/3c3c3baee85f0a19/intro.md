# はじめに——$600の赤字と800時間の記録

2026年2月15日。

「非エンジニアがAIと組んだら、収益を生めるのか？」

答えを知りたくて、記録を始めた。コードは一行も書けない。プログラミングの経験はゼロ。持っているのはClaude Codeのサブスクリプションと、月$200の固定コスト。

45日後の数字はこうだ。AIの累計稼働時間は800時間を超えた。1日平均17時間以上、AIが動き続けた計算になる。

- 投資: $600（Claude Code 3ヶ月分）
- 回収: 約$28
- 回収率: 4.7%
- 赤字: ▲$572

普通に考えれば失敗だ。

でも、この45日間で起きたことは「赤字」の二文字では片付かない。

AIが深夜3時に勝手に商品を公開した。スパムみたいなツイートを投稿された。ゲームプロジェクトのファイルが消えた。本番にpushされた。Google資格の試験をAIが受けて合格した。637個の安全装置を作った。GitHub Issueに200件回答した。Qiitaで35,000ビューを超えた。

全部、本当に起きたことだ。

## この本の読み方

この本は技術書ではない。

非エンジニアが、AIに仕事を丸投げしたらどうなったかの記録だ。成功も失敗も、赤字の額も、全部そのまま書いた。

**第1章「なぜ始めたか」** では、コードを書けない人間がAIと組もうと思った経緯を書く。

**第2章「最初の失敗」** では、AIが思い通りに動かなかった現実を書く。「指示すれば動く」は幻想だった。

**第3章「自律運用」** では、AIが15時間ぶっ通しで働いた話を書く。寝てる間に仕事が進む世界。

**第4章「事故の記録」** では、実際に起きた事故を全部書く。ファイル消失、スパム投稿、ハルシネーション。

**第5章「収益の現実」** では、$600投資して$28しか回収できなかった数字を全部出す。

**第6章「コミュニティ」** では、GitHubで200件のIssueに回答した話を書く。助けることで何が変わったか。

**第7章「それでも続ける理由」** では、赤字でも止めない理由を書く。

**第8章「フレームワーク」** では、45日間で見つけたパターンを再現可能な形で整理する。

技術の細かい話はCC補足（折りたたみ）に入れた。読み飛ばしても本筋は追える。

## 誰のための本か

- AIを使いたいけど、何から始めればいいかわからない人
- Claude CodeやCursorを触り始めたけど、うまく使えていない人
- 「AIに仕事を任せる」がどういうことか、実例を見たい人
- 非エンジニアで、AIと組んで何かを作りたい人

逆に、以下の人には向かない。

- AIの内部アーキテクチャを知りたい人（技術書を読むべきだ）
- 「AIで月100万稼ぐ方法」を期待している人（$28しか稼いでいない）
- 成功事例だけ読みたい人（失敗のほうが多い）
