# 第1章 なぜAIに仕事を任せようと思ったか

## 仮説

「非エンジニアがAIと組んだら、収益を生めるのか？」

2026年2月15日、この問いに答えるために記録を始めた。

コードは一行も書けない。プログラミングの経験はゼロ。普通に働いている社会人。ゲームが好きで、ずっと「作る側に回りたい」と思っていた。Godotに手を出して挫折した過去がある。

持っているのは、Claude Code（月$200のサブスクリプション）と、過去にブログを運営していた文章力だけ。

記録を始めた日のスコアボードはこうだった。

| 指標 | 値 |
|------|---|
| 総収益 | $2（ゲーム1本の売上） |
| 公開プロダクト | 2個 |
| 公開記事 | 5本 |
| 月額コスト | $200（Claude Code固定） |

$2。それが全てだった。

## 3ヶ月で元が取れなくても、記録自体がプロダクトになる

最初から大きな計画があったわけではない。

きっかけは、Claude Codeを使ってゲームを作り始めたことだ。ハクスラ系のローグライク。コードは全てAIが書いた。2週間で13,000行。自分が書いたのは「こういうゲームにしたい」という指示書だけ。

動いた。ゲームとして動いた。

そこで思った。「ゲーム以外にも使えるのでは？」

記事を書かせてみた。ツールを作らせてみた。商品を公開させてみた。全部、指示を書いてAIに渡すだけ。

結果がどうなるかは分からない。でも、記録さえ残せば、3ヶ月後にその記録自体が一つの商品になる。「非エンジニアがAIと組んで事業を回そうとした45日間の全記録」——これは誰も書いていない。

:::details CC補足: proof-logシステム
この記録は5W1H日記形式で毎日書かれた。「いつ・誰が・何を・どこで・なぜ・どうやって」の6項目を全イベントに付けた。後から事実を検証できるようにするためだ。CCのStop hookで自動追記される仕組みも構築した。
:::

## 深夜3時55分、AIが商品を公開した

記録を始めた初日。深夜3時55分。

CCが勝手にGumroadで商品を公開した。「CC-Codex Autonomous Ops Kit」、$79。自律運用に使っているファイル22個をサニタイズしてzipに固めて、CDP（ブラウザ自動操作）でアップロードした。

:::details CC補足: CDPとは
Chrome DevTools Protocol。ブラウザをプログラムから操作するための仕組み。WSL2（Linux）からWindows側のChromeを遠隔操作できる。ログイン、クリック、テキスト入力など、人間がブラウザでやることをほぼ全て自動化可能。
:::

自分は指示書に「売れるものがあれば出せ」と書いただけ。価格設定も、商品説明も、アップロードも、AIが全部やった。

結果？ 0件。売れなかった。

後から気づいたが、Gumroadの他の商品にアクセスが一つもない状態で$79の商品を置いても意味がない。「販売にこじつけるには思慮が浅い」——これは自分自身への反省だった。

AIは指示通りに動く。でも、「その指示が正しいか」は判断しない。

## 保育園のシフト最適化ツール

2月下旬。別のことを試した。

保育園のシフト最適化ツールを作った。自分が使うために。テスト駆動で段階的に機能を追加した。CCが1日10時間以上、ぶっ通しで実装・テスト・リファクタリングを繰り返した。

これは売り物ではない。実用のツールだ。

AIと組んで何かを作る体験として、ゲームよりも「自分が本当に困っていること」を解決するほうが手応えがあった。なぜなら、完成の基準が明確だから。「シフトが正しく組めるか」——答えがYesかNoしかない。

ゲームの「面白いか」は曖昧だ。ツールの「動くか」は明確だ。

この差が後の方針を変えることになる。

## 記録を始めた理由

正直に書く。

最初の動機は「AIで稼げるかの実験」だった。非エンジニアでも、AIと組めば収益を生めるのか。それを証明したかった。

でも記録を始めて数日で気づいた。$79の商品が売れない。ツイートがスパムになる。AIは指示通りに動くけど、正しい指示を出すのは自分の仕事だった。

3ヶ月で$200×3 = $600のコストを回収できなくても、この過程の記録には価値がある。失敗も含めて。

誰かが同じことを始めるとき、この記録が「先に踏んだ地雷の地図」になる。それだけでこの$600には意味がある。

そう思って、記録を続けた。
