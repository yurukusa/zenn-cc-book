Anthropic Academyのクイズシリーズ最終回。第1回から第5回で扱った8つの「知ったつもり」を振り返り、Anthropic Academyの受け方を案内する。

## 8つの「知ったつもり」一覧

Claude Code を140時間以上動かしてきた。APIも叩いた。RAGも組んだ。MCPサーバーも作った。

それなりに分かっているつもりだった。

Anthropic Academy（Anthropicが公式に提供している無料の認定コース群）を全12コース受けてみた。クイズで間違えた問題を見返したら、「知ったつもり」が8つ見つかった。

どれもエラーにはならない。ただ静かに、コストが増えたり、品質が落ちたりする種類の間違いだ。

### ① プロンプトキャッシュの最小トークン数は1024（[第1回](https://zenn.dev/yurukusa/articles/275d04372ad2b3)）

500トークンのプロンプトに`cache_control`を付けていた。**1024トークン未満だとキャッシュは無視される。** エラーは出ない。黙って素通りする。

キャッシュヒット時は入力トークン料金の10%（90%削減）。書き込み時は125%（初回25%割高）。最小サイズを満たさないと、書き込みコストだけ払って削減を受けられない。

### ② キャッシュブレークポイントは最後のtool定義に置く（[第1回](https://zenn.dev/yurukusa/articles/275d04372ad2b3)）

system promptだけにブレークポイントを置いていた。15個のtool定義が毎回再処理されていた。

**正解はtools配列の最後のtool定義に`cache_control`を付与すること。** system prompt + 全tool定義がまとめてキャッシュされる。最大4つのブレークポイントを設定でき、複数段階でキャッシュすれば一部が変わっても残りのキャッシュは有効。

### ③ Extended Thinkingは2ブロック返る（[第2回](https://zenn.dev/yurukusa/articles/a2dfda43cdf227)）

普通のレスポンスと同じように扱っていた。実際は**thinking block（内部推論）+ text block（最終回答）**の2つが別々に返ってくる。

thinking blockには署名（signature）が付与され改ざんを防ぐ。redacted thinking（墨消し思考）というモードもあり、安全システムがフラグした場合にthinking textが暗号化される。budget_tokensの最小値は1024トークン、max_tokensはbudget_tokensより大きくなければならない。

### ④ re-rankingは「並べ替え」ではなく「再評価」（[第3回](https://zenn.dev/yurukusa/articles/02f570b746c00f)）

「検索結果のスコア順に並べ替えること」だと思っていた。**re-rankingは別のモデルが初期検索結果をクエリと照合し直すステップ。** 別のモデル、別のスコアリング、別の目的。

初期検索はrecall重視（広く拾う）、re-rankingはprecision重視（絞り込む）。hybrid検索（ベクトル + BM25）→ マージ → re-rankingの3段パイプラインが推奨。

### ⑤ MCP Inspectorがある（[第4回](https://zenn.dev/yurukusa/articles/f84c954b6d7023)）

MCPツールのテストは「Claude Desktopにつないで試す」ものだと思っていた。`npx @modelcontextprotocol/inspector`でデバッグUIが起動する。サーバー単体でツールの動作確認ができる。

REST APIのテストにPostmanを使うのと同じ発想。クライアントに接続する前に、まずInspectorで単体テスト。これでバグの切り分けが圧倒的に楽になる。

### ⑥ ListToolsRequestでツールを発見する（[第4回](https://zenn.dev/yurukusa/articles/f84c954b6d7023)）

MCPクライアントがサーバーのツールをどう発見するか考えたことがなかった。**ListToolsRequest**というプロトコルメッセージがその仕組みだ。

レスポンスにはツールのname、description、inputSchemaが含まれる。Claudeはdescriptionを読んでツールの使い方を判断する。曖昧なdescription = 誤用。具体的なdescription = 正しい利用。

### ⑦ Workflow ≠ Agent（[第5回](https://zenn.dev/yurukusa/articles/fd2e12d180803b)）

全部「Agent」と呼んでいた。Anthropicの定義では、**Workflow = コードが次のステップを決める（固定制御フロー）**、**Agent = LLMが次のステップを決める（動的制御フロー）**。

Workflowは5パターン: 直列、分岐、並列、委任、改善ループ。Agentよりもテスト容易・精度高・予測可能。**まずWorkflowで実装可能かを先に検討し、Agentは最終手段。**

### ⑧ Temperatureのデフォルトは1.0（[第5回](https://zenn.dev/yurukusa/articles/fd2e12d180803b)）

「0.5くらいだろう」と思っていた。**デフォルトは1.0。** 最大の創造性。分類タスクや事実抽出では明示的に`temperature: 0`を設定すべき。Extended Thinking有効時は1.0固定で変更不可。

## 共通する落とし穴のパターン

8つの「知ったつもり」を並べてみると、共通のパターンが見える。

**パターン1: エラーが出ない**

8つ全てに共通する。キャッシュの最小サイズ未満でもエラーは出ない。Extended Thinkingの2ブロックを無視してもパーサーがたまたま動く。Temperature 1.0で分類タスクを回してもそれっぽい結果は返る。

エラーが出ないから気づかない。気づかないからコストが増え、品質が落ちる。

**パターン2: 「動いている」≠「正しい」**

自分のコードは動いていた。だがそれは「正しく動いている」のではなく「たまたま動いている」だった。Extended Thinkingのレスポンス処理がその典型例だ。

**パターン3: 公式ドキュメントを読んだだけでは足りない**

ドキュメントには書いてある。だが800ページのドキュメントの中から「キャッシュの最小サイズは1024トークン」を見つけて記憶している人はどれだけいるか。Academyのクイズ形式は、この種の「見落とし」を効率的にあぶり出す。

## Anthropic Academyの受け方

ここから → [anthropic.skilljar.com](https://anthropic.skilljar.com)

全12コース、全て無料、全てクイズ付き。認定は全て検証可能（認定バッジが発行される）。

### おすすめの受講順

**1. まず受けるべき3コース:**

| コース | 対象者 | 得られるもの |
|--------|--------|-------------|
| Building with Claude's API | APIを使っている人 | キャッシュ、streaming、Extended Thinking、tool useの盲点発見 |
| RAG with Claude | RAGを組んでいる人 | re-ranking、contextual embedding、hybrid検索の理解 |
| Introduction to MCP + Advanced Topics | MCPサーバーを作っている人 | Inspector、プロトコル内部、description設計 |

**2. 次に受けるべき:**

| コース | 対象者 | 得られるもの |
|--------|--------|-------------|
| Agents with Claude | AIアプリケーション設計者 | Workflow vs Agentの明確な線引き、5つのWorkflowパターン |
| Features of Claude | Claude全般のユーザー | Temperatureほか、見落としがちなAPIパラメータの理解 |

**3. 基礎固めに:**

残りのコースは基礎〜中級向け。Claude初心者なら最初から順に受けるのもいい。経験者は上記5コースを優先して、残りは必要に応じて。

### クイズの特徴

- 暗記ではなく理解を問う形式
- 「正しい選択肢を選べ」ではなく「この状況で最適なアプローチは」が多い
- 経験者ほど「知ったつもり」で間違える
- 不合格でも何度でも再受験可能

:::details CC補足: Anthropic Academyの技術的な位置づけ
Anthropic Academyは2025年に開始されたAnthropicの公式教育プログラム。Skilljar（SaaS型のeラーニングプラットフォーム）上でホストされている。

コースは大きく3つのレベルに分かれる:
1. **入門**: Claude の基本概念、プロンプトエンジニアリング
2. **中級**: API利用、RAG、MCP
3. **上級**: Agent設計、高度なワークフローパターン

各コースは動画レッスン + テキスト + ハンズオン + クイズで構成される。クイズの合格が認定の条件。

認定バッジはLinkedIn等のプロフィールに掲載可能。2026年3月時点では全コース無料で提供されている。
:::

## 最後に

Academyを受ける前と後で、自分のAPIの使い方が変わった。

- キャッシュブレークポイントを最後のtool定義に移動した
- Extended Thinkingのパーサーを2ブロック対応に修正した
- RAGパイプラインにre-rankingステップを追加した
- MCPサーバーのテストフローにInspectorを組み込んだ
- 分類タスクのTemperatureを0に設定した
- 「Agent」と呼んでいたものの大半を「Workflow」と認識し直した

どれも小さな修正だ。だがAPIを毎日叩いている以上、小さな修正の積み重ねが月単位のコストと品質に効いてくる。

8つの「知ったつもり」は、APIを何ヶ月も使ってきた自分だからこそ発生した盲点だ。初心者にはそもそも盲点が生まれない（知らないことは知っているから）。中級者こそ受ける価値がある。

分かったつもりの人間ほどよく間違える。自分がその証拠だ。
