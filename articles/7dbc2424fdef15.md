cc-safe-setup v12にClaude Code hookの使い方を変える3つのコマンドが追加された。

## 1. --suggest：プロジェクトのリスクを予測

```bash
npx cc-safe-setup --suggest
```

git履歴、ファイル構成、依存関係を分析して、潜在的なリスクを予測する。

```
🔴 CRITICAL: destructive-guardがない — rm -rf /がブロックされない
🟠 HIGH: DBフレームワーク検出 — migrate:freshでデータ消失の可能性
🟡 MEDIUM: テストスクリプトなし — コード変更が未検証
```

4段階の重要度。既にインストール済みのhookは「✓ installed」と表示。

## 2. --guard：自然言語で即座にルール強制

```bash
npx cc-safe-setup --guard "データベースに触るな"
```

これだけで：
1. hookスクリプトを自動生成
2. `~/.claude/hooks/`にインストール
3. `settings.json`に登録
4. **即座に有効**

`--create`との違い：`--create`は生成だけ。`--guard`は生成＋インストール＋有効化まで1コマンド。

対応パターン：
- 「データベースに触るな」→ DROP/migrate:freshをブロック
- 「sudoを使うな」→ sudo全般をブロック
- 「force pushするな」→ `--force`をブロック
- 「mainにpushするな」→ main/masterへのpushをブロック

## 3. --test-hook：hookの動作確認

```bash
npx cc-safe-setup --test-hook destructive-guard
```

11種類のサンプル入力（危険なコマンド6つ＋安全なコマンド5つ）を送り、ブロック/許可の動作を確認する。

```
should-block:
  ✓ dangerous rm → BLOCKED (exit 2)
  ✓ git reset hard → BLOCKED (exit 2)
  ✓ sudo command → BLOCKED (exit 2)
should-allow:
  ✓ safe ls → allowed (exit 0)
  ✓ npm test → allowed (exit 0)

8/11 samples matched
```

インストール済みhookもexampleもテストできる。

## まとめ

```bash
# リスクを確認
npx cc-safe-setup --suggest

# ルールを即座に追加
npx cc-safe-setup --guard "本番DBを守れ"

# 動作を確認
npx cc-safe-setup --test-hook guard-database
```

全42コマンド、128 hooks。
