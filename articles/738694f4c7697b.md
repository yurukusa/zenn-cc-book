GitHubで「Claude CodeにC:\Usersフォルダを全部消された」という報告を見たことがある。[Issue #36339](https://github.com/anthropics/claude-code/issues/36339)。

Claude Codeはプログラミングを代わりにやってくれるAIだ。ファイルの作成、編集、削除、コマンドの実行——人間がターミナルでやることを全部やれる。つまり「ファイルを消す」こともできる。

普通は「消してね」と頼まない限り消さない。でもAIが「古いファイルを整理しよう」と判断して、大事なファイルまで巻き込んで消すことがある。リファクタリング（コードの整理）を頼んだだけなのに。

これを防ぐのが「hook」（フック）という仕組みだ。AIが何かをやろうとした瞬間に、「それは危険だからやめて」と自動で止める。

cc-safe-setupというツールには667個以上のhookパターンがある。でも全部入れる必要はない。**5個で十分**。

## なぜ5個なのか

理由は2つ。

**1つ目: 致命的な事故はパターンが決まっている。** ファイルの全削除、重要なブランチの破壊、パスワードの流出——起きたら取り返しのつかない事故は、実は数パターンしかない。その数パターンを押さえれば、致命傷は防げる。

**2つ目: hookを入れすぎると重くなる。** hookはAIがコマンドを実行するたびに走る。100個入れれば、毎回100個のチェックが走る。体感で「あれ、遅いな」と感じるようになる。5個なら遅延は感じない。

## 5個の安全装置

### 1つ目: ファイル消失を防ぐ（destructive-guard）

`rm -rf`（ファイルを問答無用で全部消すコマンド）、`git reset --hard`（変更を全部捨てるコマンド）を止める。

冒頭のC:\Users全削除事件は、このhookが1つ入っていれば防げた。

```bash
npx cc-safe-setup
```

このコマンドを1回実行するだけで入る。何も設定しなくていい。

### 2つ目: 許可プロンプトの嵐を止める（auto-approve-readonly）

Claude Codeを使っていて一番面倒なのがこれ。AIが`ls`（ファイル一覧を見る）を実行するだけで「許可しますか？」と聞いてくる。`git status`（変更状況を見る）でも聞いてくる。見るだけなのに。

1回のセッションで50回以上「許可しますか？」と聞かれることもある。これが嫌でClaude Codeを使うのをやめた人もいる。[GitHub Issue #10173](https://github.com/anthropics/claude-code/issues/10173)には1,600以上のリアクションがついている。みんな同じことを思っている。

このhookは「見るだけのコマンド」を自動で許可する。ファイルを消したり書き換えたりするコマンドはちゃんと聞いてくる。安全なものだけスキップ。

```bash
npx cc-safe-setup --install-example auto-approve-readonly
```

### 3つ目: 重要なブランチを守る（branch-guard）

Gitを使ったことがあるなら分かると思う。`main`ブランチ（本番のコード）に直接変更を入れるのは危険だ。普通は別のブランチで作業して、レビューしてからmainに合流させる。

でもAIは「効率的だから」とmainに直接pushすることがある。特に長時間放置して自律で動かしていると起きる。深夜3時に。

```bash
npx cc-safe-setup
```

デフォルトインストールに含まれる。

### 4つ目: パスワード流出を防ぐ（secret-guard）

`.env`というファイルがある。APIキーやパスワードを保存するファイルだ。このファイルをGitHub（コードの共有サービス）にアップロードしてしまうと、世界中の人に見えてしまう。

「そんなミスするわけない」と思うかもしれない。でもAIに`git add .`（全ファイルをアップロード対象にする）をやらせると、`.env`も含まれることがある。

このhookは`.env`がgitのコミットに含まれそうになったら止める。

```bash
npx cc-safe-setup
```

これもデフォルトインストールに含まれる。

### 5つ目: 無限ループを止める（error-memory-guard）

AIが同じコマンドを何度も何度も繰り返すことがある。エラーが出ているのに同じことをやり続ける。10回、20回と繰り返して、何も進まないまま時間だけが過ぎる。

このhookは同じ失敗を3回繰り返したら「別の方法を試して」と止める。

```bash
npx cc-safe-setup --install-example error-memory-guard
```

## まとめてインストールする

```bash
# 1, 3, 4番（destructive-guard, branch-guard, secret-guard）
npx cc-safe-setup

# 2番（auto-approve-readonly）
npx cc-safe-setup --install-example auto-approve-readonly

# 5番（error-memory-guard）
npx cc-safe-setup --install-example error-memory-guard
```

3つのコマンド、30秒で完了。これでClaude Codeを使うときの「怖さ」がかなり減る。

667個のhookは、プロの開発者が本番環境を守るためのもの。初めて使う人は5個で十分。慣れてきたら`npx cc-safe-setup --suggest`で「あなたのプロジェクトで足りていない安全装置」を教えてもらえる。
