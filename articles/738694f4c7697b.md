cc-safe-setupには175個のhookがある。全部は入れなくていい。5個で十分。

## 1. auto-approve-readonly

```bash
npx cc-safe-setup --install-example auto-approve-readonly
```

**効果**: パーミッションプロンプト80%削減。

`cat`、`ls`、`grep`、`git status`を毎回承認する苦痛から解放される。これが一番体感が変わる。

## 2. destructive-guard

```bash
npx cc-safe-setup
```

**効果**: `rm -rf /`、`git reset --hard`をブロック。

入れなかった人の話 → [GitHub #36339](https://github.com/anthropics/claude-code/issues/36339)（C:\Usersフォルダ全削除）

## 3. branch-guard

```bash
npx cc-safe-setup
```

**効果**: mainへのpush、force-pushをブロック。

自律セッションの深夜3時にmainが壊されるのを防ぐ。

## 4. secret-guard

```bash
npx cc-safe-setup
```

**効果**: `.env`のgit commitを防止。

`git add .`で.envが公開されてAPIキーが漏洩 — これを1行で防ぐ。

## 5. error-memory-guard

```bash
npx cc-safe-setup --install-example error-memory-guard
```

**効果**: 失敗コマンドのリトライを3回で停止。

Claudeが`npm install broken-pkg`を10回繰り返す問題を解決。「別のアプローチを試せ」と表示される。

## インストール

```bash
# 上の5個を1コマンドで
npx cc-safe-setup --shield
```

`--shield`なら上の5個＋スタック検出＋推奨hookが全部入る。

## もっと知りたい

- 全175hookの一覧: `npx cc-safe-setup --examples`
- リスク予測: `npx cc-safe-setup --suggest`
- 自然言語でルール追加: `npx cc-safe-setup --guard "DBに触るな"`
