## この記事について

プログラミング経験がなくても、Claude Codeを使えばアプリケーションを作れる。この記事では、環境構築から始めて、実際に動くPythonスクリプトを作るまでを一歩ずつ解説する。

筆者のゆるくさもプログラミング未経験から始めて、Claude Codeだけで13,000行のゲームを完成させた。この記事はその経験を踏まえた実践的な入門ガイドである。

## Claude Codeとは何か

Claude Codeは、Anthropicが提供するCLI（コマンドラインインターフェース）ツールだ。ターミナル上で動作し、AIと対話しながらコードを書いたり、ファイルを操作したり、コマンドを実行したりできる。

一般的なチャットAI（ChatGPT、Claude.ai等）との決定的な違いは以下の点にある。

- **ファイルシステムに直接アクセスできる** -- コードをコピペする必要がない
- **コマンドを実行できる** -- `python script.py` をAIが直接実行し、結果を確認する
- **プロジェクト全体を把握できる** -- 数百ファイルのプロジェクトでも構造を理解する
- **自律的に修正できる** -- エラーが出たら自分で原因を調べて直す

つまり、Claude Codeは「コードを書くAI」ではなく「開発を代行するAI」に近い。

## 前提条件

この記事の手順は以下の環境を前提としている。

- **OS**: Windows 10/11（WSL2を使用）
- **必要なもの**: Anthropicアカウント、Claude ProまたはMaxのサブスクリプション
- **プログラミング経験**: 不要
- **ターミナル経験**: 不要（この記事で基礎から説明する）

macOS/Linuxユーザーはステップ1（WSL2インストール）をスキップして、ステップ2から始めればよい。

## 環境構築

### Step 1: WSL2のインストール

WindowsでClaude Codeを使うにはWSL2（Windows Subsystem for Linux）が必要。これはWindows上でLinuxを動かす仕組みで、Microsoftが公式に提供している。

PowerShellを**管理者権限**で開く。スタートメニューで「PowerShell」と検索し、「管理者として実行」を選ぶ。

```powershell
wsl --install
```

これだけでUbuntuがインストールされる。再起動を求められたら再起動する。

再起動後、自動的にUbuntuのセットアップが始まる。ユーザー名とパスワードを設定する。パスワードは入力しても画面に表示されないが、ちゃんと入力されているので安心してほしい。

設定が完了したら、スタートメニューから「Ubuntu」を開く。黒い画面（ターミナル）が表示されれば成功。

#### ターミナルの基本操作

最低限知っておくべきコマンドを3つだけ紹介する。

```bash
# 今いる場所（ディレクトリ）を確認
pwd

# 今いるディレクトリのファイル一覧を表示
ls

# ディレクトリを移動する
cd ~/my-project
```

これだけ知っていれば、Claude Codeを使い始められる。他のコマンドはClaude Codeに聞けば教えてくれる。

### Step 2: Node.jsのインストール

Claude CodeはNode.js上で動作する。WSL2のターミナルで以下を実行する。

```bash
# nvm（Node Version Manager）をインストール
# Node.jsのバージョンを管理するためのツール
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash

# ターミナルを再起動するか、以下を実行して反映
source ~/.bashrc

# Node.js LTS（長期サポート）版をインストール
nvm install --lts

# 確認
node --version
# v22.x.x のような表示が出れば成功
```

エラーが出た場合、ターミナルを一度閉じて開き直すと解決することが多い。

### Step 3: Claude Codeのインストール

```bash
# Claude Codeをグローバルインストール
npm install -g @anthropic-ai/claude-code

# 確認
claude --version
# バージョン番号が表示されれば成功
```

### Step 4: 認証

Claude Codeを初めて起動すると認証が求められる。

```bash
# プロジェクトディレクトリを作成して移動
mkdir -p ~/my-first-project
cd ~/my-first-project

# Claude Codeを起動
claude
```

ブラウザが開き、Anthropicアカウントでのログインを求められる。ログインすると認証が完了し、Claude Codeが使えるようになる。

WSL2からブラウザが開かない場合は、表示されたURLをWindowsのブラウザに手動でコピペすればよい。

**料金について**: Claude Codeの利用にはClaude ProまたはMaxのサブスクリプションが必要。Proプラン（月額$20）でも使えるが、使用量に制限がある。Maxプラン（月額$100〜$200）であれば、より多くの使用が可能。入門段階ではProプラン（$20）で十分。

## 基本操作

### 対話の始め方

Claude Codeを起動すると、プロンプトが表示される。

```
~/my-first-project
>
```

ここに日本語で指示を書くだけでよい。

```
> 現在のディレクトリにあるファイルを教えて
```

Claude Codeは `ls` コマンドを実行し、結果を教えてくれる。初めてのうちは「何ができるか」を質問してみるのもよい。

### 許可の仕組み

Claude Codeがファイルを変更したりコマンドを実行したりする際、確認が表示される。

```
Claude wants to run: mkdir src
Allow? [y/n]
```

`y` を押すと実行される。安全のための仕組みだが、慣れてくると毎回確認するのが面倒になる。その場合は信頼できる操作に対して「Always allow」を選択することもできる。

### 便利なコマンド

Claude Codeの対話中に使えるショートカット。

```
/help       -- ヘルプを表示
/clear      -- 会話履歴をクリア
/compact    -- 会話を要約して圧縮（トークン節約）
Ctrl+C      -- 現在の操作をキャンセル
```

## 最初のプロジェクト: 家計簿スクリプト

実際にClaude Codeを使って簡単なPythonスクリプトを作ってみよう。

### プロジェクトの作成

Claude Codeに以下のように入力する。

```
> Pythonで簡単な家計簿スクリプトを作ってほしい。
> 機能:
> - 収入と支出を記録できる
> - CSVファイルにデータを保存する
> - 月ごとの合計を表示できる
> - コマンドラインで操作する
```

Claude Codeはこの指示を受けて、以下のような動きをする。

1. Pythonファイルを作成する
2. 必要な機能を実装する
3. 動作確認のためにスクリプトを実行する

完成したスクリプトの例:

```python
#!/usr/bin/env python3
"""
家計簿スクリプト
- 収入/支出をCSVに記録
- 月別サマリーを表示
"""
import csv
import os
from datetime import datetime

DATA_FILE = "kakeibo.csv"

def init_file():
    """CSVファイルが存在しない場合、ヘッダー行を作成する"""
    if not os.path.exists(DATA_FILE):
        with open(DATA_FILE, "w", newline="", encoding="utf-8") as f:
            writer = csv.writer(f)
            writer.writerow(["日付", "種類", "カテゴリ", "金額", "メモ"])

def add_record(record_type, category, amount, memo=""):
    """収入または支出を1件記録する"""
    init_file()
    with open(DATA_FILE, "a", newline="", encoding="utf-8") as f:
        writer = csv.writer(f)
        writer.writerow([
            datetime.now().strftime("%Y-%m-%d"),
            record_type,
            category,
            amount,
            memo
        ])
    print(f"記録しました: {record_type} {category} {amount}円")

def show_summary():
    """月ごとの収支サマリーを表示する"""
    init_file()
    monthly = {}
    with open(DATA_FILE, "r", encoding="utf-8") as f:
        reader = csv.DictReader(f)
        for row in reader:
            month = row["日付"][:7]  # YYYY-MM
            if month not in monthly:
                monthly[month] = {"収入": 0, "支出": 0}
            monthly[month][row["種類"]] += int(row["金額"])

    for month, data in sorted(monthly.items()):
        balance = data["収入"] - data["支出"]
        print(f"{month}: 収入 {data['収入']:,}円 / "
              f"支出 {data['支出']:,}円 / 残高 {balance:+,}円")

def main():
    """メインメニュー"""
    init_file()
    while True:
        print("\n=== 家計簿 ===")
        print("1. 収入を記録")
        print("2. 支出を記録")
        print("3. 月別サマリー")
        print("4. 終了")

        choice = input("選択: ")
        if choice == "1":
            cat = input("カテゴリ（例: 給料）: ")
            amt = input("金額: ")
            memo = input("メモ（省略可）: ")
            add_record("収入", cat, int(amt), memo)
        elif choice == "2":
            cat = input("カテゴリ（例: 食費）: ")
            amt = input("金額: ")
            memo = input("メモ（省略可）: ")
            add_record("支出", cat, int(amt), memo)
        elif choice == "3":
            show_summary()
        elif choice == "4":
            print("終了します")
            break

if __name__ == "__main__":
    main()
```

### 実行してみる

Claude Codeに「実行して」と言えば動かしてくれるが、自分でも実行できる。

```bash
python3 kakeibo.py
```

### 改善を依頼する

動いたら、さらに機能追加を依頼できる。

```
> カテゴリ別の集計機能も追加してほしい
```

```
> データをグラフで表示する機能を追加してほしい。matplotlibを使って
```

このように、対話しながら少しずつ育てていくのがClaude Codeでの開発スタイルだ。

## 指示の出し方のコツ

Claude Codeの出力品質は、入力する指示の具体性に大きく左右される。

### 悪い指示の例

```
> いい感じのアプリ作って
```

「いい感じ」はAIにはわからない。何が欲しいのか具体的にする必要がある。

### 良い指示の例

```
> Pythonで天気予報を取得するスクリプトを作ってほしい。
> - OpenWeatherMap APIを使う
> - 都市名を引数で受け取る
> - 気温、湿度、天気の説明を日本語で表示する
> - APIキーは環境変数WEATHER_API_KEYから読む
```

**ポイント:**
- 使う言語・ライブラリを指定する
- 機能を箇条書きで列挙する
- 入出力の形式を明確にする
- 制約条件があれば書く

### 段階的に依頼する

一度に全部を求めるのではなく、小さなステップで進めるのも有効。

```
1回目: 「まず基本的なCLIの骨格だけ作って」
2回目: 「CSV保存機能を追加して」
3回目: 「月別集計機能を追加して」
4回目: 「入力バリデーションを追加して」
```

これなら、各ステップで動作確認してから次に進める。

## ファイル管理とGit

### なぜGitが必要か

Claude Codeはファイルを直接編集する。うまくいかない変更をされた場合、元に戻す手段が必要。それがGitだ。

Gitは「ファイルの変更履歴を記録するツール」と思えばよい。ゲームのセーブポイントのようなもの。

### 最低限のGit操作

```bash
# プロジェクトでGitを初期化（最初の1回だけ）
git init

# 全ファイルを記録対象に追加
git add .

# セーブポイントを作成（コミット）
git commit -m "家計簿スクリプトの初版を作成"
```

これだけで、いつでもこの時点の状態に戻れるようになる。

Claude Codeに「Gitで管理して」と言えば、これらの操作を代行してくれる。実際の開発では、以下のように依頼するとよい。

```
> 今の状態をGitにコミットしてほしい
```

### 元に戻したい場合

Claude Codeが意図しない変更をした場合、Gitで戻せる。

```
> 直前のコミットの状態に戻してほしい
```

自分で操作する場合は以下のコマンド。

```bash
# 直前のコミットとの差分を確認
git diff

# 変更を全て取り消して直前のコミットに戻す
git checkout .
```

## CLAUDE.md: プロジェクトの説明書

プロジェクトのルートに `CLAUDE.md` というファイルを置くと、Claude Codeが毎回それを読んでプロジェクトの文脈を理解する。

```markdown
# 家計簿プロジェクト

## 概要
Pythonで作ったCLI家計簿アプリ

## ファイル構成
- kakeibo.py -- メインスクリプト
- kakeibo.csv -- データファイル（Git管理しない）

## ルール
- データファイル(csv)はgitignoreに入れる
- 日本語でコメントを書く
- Python 3.11以上を前提とする
```

`CLAUDE.md` があることで、Claude Codeが毎回「このプロジェクトは何か」を質問する必要がなくなる。新しいセッションでも、前回の文脈を引き継いで作業できる。

これは非エンジニアにとって特に重要で、「前回何をお願いしたか」をAIが覚えてくれる仕組みだと思えばよい。

## よくあるトラブルと対処法

### 「Permission denied」が出た

ファイルの書き込み権限がない場合に発生する。

```
> このエラーが出た: Permission denied
```

とClaude Codeに伝えれば、`chmod` コマンドで権限を修正してくれる。

### 「ModuleNotFoundError」が出た

Pythonのライブラリが未インストールの場合に発生する。

```
> このエラーが出た:
> ModuleNotFoundError: No module named 'matplotlib'
```

Claude Codeが `pip install matplotlib` を実行して解決してくれる。

### AIの応答が途中で止まった

長い処理中にタイムアウトすることがある。

```
> 続きをやって
```

で再開できる。

### AIが同じエラーをループしている

稀にAIが同じ修正を繰り返してしまうことがある。

```
> 別のアプローチで解決してほしい。今までの方法はうまくいっていない。
```

明確に「別の方法で」と指示すると、違う角度から問題に取り組んでくれる。

## よくある質問

### Q. プログラミングを全く知らなくても使えますか？

使える。ただし、以下の最低限の知識があると効率が上がる。

- **ターミナルの基本操作** -- `cd`（移動）、`ls`（一覧）、`pwd`（現在地）
- **ファイルとディレクトリの概念** -- フォルダ構造が理解できること
- **テキストエディタの使い方** -- VS Code等でファイルを開いて中身を確認できること

これらはClaude Codeに「ターミナルの基本を教えて」と聞けば教えてくれる。

### Q. どの程度のものが作れますか？

Claude Codeの能力は非常に高い。実際にプログラミング未経験者が作った例として、以下がある。

- **13,000行のローグライクRPG** -- 装備システム、ダンジョン生成、戦闘AI、セーブ/ロード
- **ブラウザ自動操作ツール** -- Chrome DevTools Protocol連携
- **ダンジョン経営シミュレーション** -- Godot 4.3で開発、27回のメジャーアップデート

重要なのは「一度に全部作ろうとしない」こと。小さく始めて、対話しながら育てていく。

### Q. 料金はどのくらいかかりますか？

| プラン | 月額 | Claude Code使用 |
|--------|------|----------------|
| Claude Pro | $20 | 可能（制限あり） |
| Claude Max 5X | $100 | 実用的 |
| Claude Max 20X | $200 | ヘビーユース向け |

入門段階ではProプラン（$20）で十分。制限に引っかかるようになったらアップグレードを検討すればよい。

### Q. ChatGPTとどう違いますか？

ChatGPTはWebブラウザの中で動く「対話型AI」。Claude Codeはターミナルで動く「開発代行AI」。

ChatGPTは質問に答えてくれるが、あなたのPCのファイルを直接触ることはできない。Claude Codeは、あなたのPCの中で直接ファイルを作り、コードを実行し、Gitで管理してくれる。

「コードを教えてもらう」ならChatGPT。「コードを書いてもらう」ならClaude Code。

## まとめ

- Claude Codeは「コードを書くAI」ではなく「開発を代行するAI」
- 環境構築はWSL2 + Node.js + `npm install -g @anthropic-ai/claude-code` の3ステップ
- まずは簡単なPythonスクリプトから始めて、対話しながら育てる
- Gitで変更履歴を管理し、`CLAUDE.md` でプロジェクトの文脈を共有する
- プログラミングの知識は不要だが、「何を作りたいか」の言語化能力は必要
- エラーが出てもClaude Codeに丸投げすれば大抵解決する

非エンジニアであることは、もはやモノづくりのハードルにはならない。必要なのは「何を作りたいか」というビジョンだけだ。
