Claude Codeを長時間使っていると、ある時点で「なんか回答がおかしい」「さっき共有したコードを忘れている」と感じることがある。

その原因の多くは**コンテキストウィンドウの枯渇**だ。

## コンテキストウィンドウとは

Claude Sonnetは20万トークンのコンテキストウィンドウを持つ。会話が長くなるにつれて、古いメッセージは徐々に「見えなく」なる。指示した制約、読み込んだファイル、説明したコードベース——これらが静かに消えていく。

問題は、**今どのくらい使っているかが分からない**こと。

## cc-context-check

そこで作ったのが `cc-context-check`。

```bash
npx cc-context-check
```

```

Context limit: 200.0k tokens (Claude Sonnet/Opus)

   ████████████░░░░░░░░░░░░░░░░░░ 40.1% used
   80.2k input · 1.2k output · 119.8k remaining

🟡 ~/                   [b7d4] 2h ago · 5.9 MB
   █████████████████████░░░░░░░░░ 71.5% used
   143.0k input · 89 output · 57.0k remaining

```

- 🟢 70%未満：余裕あり
- 🟡 70〜85%：そろそろ `/compact` を検討
- 🔴 85%以上：今すぐ `/compact`

## 仕組み

Claude Codeは `~/.claude/projects/` にセッションをjsonl形式で保存している。各アシスタントメッセージには `usage` フィールドが含まれる：

```json
{
  "usage": {
    "input_tokens": 1,
    "cache_read_input_tokens": 79927,
    "cache_creation_input_tokens": 917,
    "output_tokens": 158
  }
}
```

**実際のコンテキスト使用量** = `input_tokens + cache_read_input_tokens + cache_creation_input_tokens`

`input_tokens: 1` という値は誤解を招くが、実際の文脈は `cache_read_input_tokens`（キャッシュから読んだトークン）に入っている。この合計値を200,000で割ることで使用率が分かる。

## オプション

```bash
npx cc-context-check --all    # 上位20セッションを表示（デフォルトは5）
npx cc-context-check --json   # JSONで出力（スクリプト連携用）
```

## インストール不要

`npx` で即実行。ローカルの `~/.claude/` を読むだけなので外部通信なし。
