## 許可プロンプトがうるさい

Claude Codeを自律で回していると、`git log`を実行するだけで許可を求められる。`cd /path && git diff`で許可を求められる。`ls`で許可を求められる。

読み取り専用のコマンドなのに、毎回「Allow?」と聞かれる。自律運用の最大の障害。

settings.jsonのallowlistに書いても、Claudeがコメント行（`# Check the status`）を付けるとパターンが一致しなくなり、結局許可を求められる（[GitHub #29582](https://github.com/anthropics/claude-code/issues/29582), 18 reactions）。

## PermissionRequestフックという選択肢

PreToolUseフックはコマンドの実行を制御できるが、許可ダイアログ自体は表示される。

**PermissionRequestフック**は許可ダイアログが表示される前に介入できる。`permissionDecision: "allow"`を返せば、ユーザーには何も表示されない。

```json
{
  "hooks": {
    "PermissionRequest": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "~/.claude/hooks/auto-approve-readonly.sh"
          }
        ]
      }
    ]
  }
}
```

## 読み取り専用コマンドだけ自動承認する

```bash
#!/bin/bash
INPUT=$(cat)
COMMAND=$(echo "$INPUT" | jq -r '.tool_input.command // empty')
[[ -z "$COMMAND" ]] && exit 0

# cd + コマンドの場合、実際のコマンドを抽出
ACTUAL=$(echo "$COMMAND" | sed 's/^\s*cd\s\+[^&]*&&\s*//')
BASE=$(echo "$ACTUAL" | awk '{print $1}')

case "$BASE" in
    # 読み取り専用のgit操作
    git)
        SUB=$(echo "$ACTUAL" | awk '{print $2}')
        case "$SUB" in
            log|diff|status|branch|show|tag|remote|rev-parse)
                jq -n '{hookSpecificOutput:{hookEventName:"PermissionRequest",permissionDecision:"allow",permissionDecisionReason:"read-only git"}}'
                exit 0 ;;
        esac ;;
    # 読み取り専用のファイル操作
    ls|cat|head|tail|wc|file|stat|which|echo|pwd|date)
        jq -n '{hookSpecificOutput:{hookEventName:"PermissionRequest",permissionDecision:"allow",permissionDecisionReason:"read-only command"}}'
        exit 0 ;;
    # 読み取り専用の検索
    grep|rg|find|fd|tree)
        # -deleteや-exec rmが含まれていなければ安全
        if ! echo "$ACTUAL" | grep -qE '\-delete|\-exec\s+rm'; then
            jq -n '{hookSpecificOutput:{hookEventName:"PermissionRequest",permissionDecision:"allow",permissionDecisionReason:"read-only search"}}'
            exit 0
        fi ;;
esac

exit 0  # 判断しない = 通常の許可フローに任せる
```

## PreToolUseとの違い

| | PreToolUse | PermissionRequest |
|---|---|---|
| タイミング | ツール実行前 | 許可ダイアログ表示前 |
| 許可ダイアログ | 表示される | 表示されない |
| exit 2の効果 | ツール実行をブロック | 許可を拒否 |
| `permissionDecision` | allow/deny/ask | allow/deny/ask |
| 主な用途 | セキュリティチェック | UX改善（不要な許可を消す） |

## まとめ

PermissionRequestフックは「許可プロンプトを消す」ために設計されている。読み取り専用コマンドに対して`permissionDecision: "allow"`を返すだけで、自律運用中の中断が大幅に減る。

ワンコマンドでインストール:

```bash
npx cc-safe-setup
```
