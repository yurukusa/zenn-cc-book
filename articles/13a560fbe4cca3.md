## どのファイルが一番チャーンしているか？

Claude Codeを60日以上24時間動かして気づいたこと: 特定のファイルが何度も何度も書き直されている気がする。でも「気がする」だけで数値がない。

実際に集計してみると、1位のファイルは**合計933回**ツールから触られていた（書き込み669回、読み取り264回）。

これを可視化するツールを作った。

## cc-file-churn

```bash
npx cc-file-churn
```

`~/.claude/` の全セッションログを走査して、ファイルごとの活動頻度をランキング表示する。

```
cc-file-churn — top 10 files by total activity (all time)

#1  ~/.claude/plans/staged-hatching-backus.md
    ████████████████████ 933  (w:669 r:264 s:0)

#2  ~/projects/aetheria/dungeon_game.py
    ██████████████████░░ 671  (w:301 r:290 s:80)

#3  ~/projects/cc-loop/cc-toolkit/index.html
    ████████░░░░░░░░░░░░ 312  (w:189 r:123 s:0)
...

755 session files · 2219 unique files · 12476 tool calls
```

## 何が分かるか

集計してみて意外だったのは、1位がソースコードじゃなく**プランニングドキュメント**だったこと。Claude Codeは自分のタスクプランを669回書き直し、264回読み返していた。AIが自分の作業状態を管理している様子がそのまま数値に出た。

2位は `dungeon_game.py`（ゲームの中核コード）。読み書きがほぼ同数というのが面白い。変更する前にファイルを読んでコンテキストを把握してから編集している、という行動パターンがよく見える。

3位は `cc-toolkit/index.html`（このツールコレクションのランディングページ）。書き込み多め、読み取り少なめ。ツールカードを追加するたびに更新されているから当然といえば当然。

## フィルターオプション

```bash
# 書き込み回数順（最も修正されたファイル）
npx cc-file-churn --writes

# 読み取り回数順（最も参照されたファイル）
npx cc-file-churn --reads

# 直近7日間のみ
npx cc-file-churn --days=7

# 特定プロジェクトのみ
npx cc-file-churn --project=cc-loop

# JSON出力
npx cc-file-churn --json

# 上位20件表示
npx cc-file-churn 20
```

## ブラウザ版

インストール不要でブラウザから試せる:

→ **[yurukusa.github.io/cc-file-churn](https://yurukusa.github.io/cc-file-churn/)**

`~/.claude` フォルダをドラッグ&ドロップするだけ。データはローカルでのみ処理され、どこにも送信されない。

フィルターボタン（全活動 / 最多修正 / 最多参照）付き、各ファイルのバー表示でw/r/sの内訳も見れる。

## ファイルチャーンから何を読み取るか

- **書き込みが多い設計ファイル**: 設計が固まっていない可能性
- **読み書きバランスが取れたソースファイル**: 正常なイテレーション開発
- **書き込み >> 読み取りのファイル**: パッチの連続？根本設計を見直す価値あり
- **読み取りが多い設定ファイル**: AIが頻繁に参照する「辞書」的なファイル

同じ300回でも「良いチャーン（改善の繰り返し）」と「悪いチャーン（同じ問題を何度も修正）」がある。数値を見てから判断できる。
