## CLAUDE.mdは「お願い」、Hooksは「強制」

Claude Codeに「mainにpushしないで」とCLAUDE.mdに書いた。
コンテキストが80%埋まった頃、Claudeは堂々とmainにpushした。

CLAUDE.mdはプロンプトに含まれるテキスト。コンテキストが埋まると優先度が下がる。モデルは「忘れる」のではなく「相対的に軽視する」。

Hooksは違う。プロセスレベルで強制される。モデルが何を考えていようが、`exit 2`を返せば操作はブロックされる。

## Hooksの仕組み（3行で理解）

```
1. Claudeがツール（Bash, Edit, Write）を呼ぶ
2. 実行前にシェルスクリプトが走る（PreToolUse）
3. スクリプトがexit 2を返すと、操作がブロックされる
```

これだけ。フレームワーク不要。依存ゼロ。bashとjqだけ。

## 最小限のhook

```bash
#!/bin/bash
# ~/.claude/hooks/guard.sh
COMMAND=$(cat | jq -r '.tool_input.command // empty')
[ -z "$COMMAND" ] && exit 0

if echo "$COMMAND" | grep -qE 'rm\s+.*-rf\s+/'; then
  echo "BLOCKED: rm -rf on root" >&2
  exit 2  # ← ブロック
fi
exit 0    # ← 許可
```

## settings.jsonへの登録

```json
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "command",
        "command": "bash ~/.claude/hooks/guard.sh"
      }]
    }]
  }
}
```

`matcher`はツール名の正規表現。`"Bash"`はBashコマンドだけ。`"Edit|Write"`はファイル編集。`""`は全ツール。

## 4つのイベント

| イベント | タイミング | 用途 |
|---------|-----------|------|
| `PreToolUse` | ツール実行**前** | ブロック、承認 |
| `PostToolUse` | ツール実行**後** | 結果チェック、監視 |
| `Stop` | Claude応答完了時 | 通知、クリーンアップ |
| `SubagentStop` | サブエージェント完了時 | エージェント監視 |

## exit codeの意味

| コード | 効果 |
|--------|------|
| `0` | 許可（デフォルト） |
| `2` | **ブロック**（モデルがバイパス不可） |
| `1` | hookエラー（無視される） |

**stdout JSON**で決定を上書きもできる：

```bash
# 自動承認
echo '{"decision":"approve","reason":"Safe command"}'
exit 0
```

## ワンコマンドで8つの安全hookを入れる

```bash
npx cc-safe-setup
```

これで以下が一発で入る：
- `rm -rf /` ブロック
- main/masterへのpush ブロック
- `.env`のgit add ブロック
- 構文エラー検出
- コンテキスト残量警告
- bashコメント修正
- cd+git自動承認
- APIエラー通知

## よく使うパターン3つ

### 1. 特定コマンドをブロック

```bash
npx cc-safe-setup --install-example block-database-wipe
npx cc-safe-setup --install-example no-sudo-guard
```

### 2. 安全なコマンドを自動承認

```bash
npx cc-safe-setup --install-example auto-approve-python
npx cc-safe-setup --install-example auto-approve-docker
```

### 3. 自然言語でカスタムhookを生成

```bash
npx cc-safe-setup --create "npm publishの前にテストを実行したか確認"
```

## CLAUDE.md vs Hooks：使い分け

**CLAUDE.mdが得意なこと：**
- コードスタイル、命名規則
- 設計方針、アーキテクチャ判断
- コミットメッセージのフォーマット

**Hooksが得意なこと：**
- 危険なコマンドのブロック
- パーミッション制御
- コスト監視、セッション管理

両方使うのがベスト。CLAUDE.mdで方針を伝え、Hooksで安全を強制する。
