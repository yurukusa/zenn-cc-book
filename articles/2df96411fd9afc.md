Claude Codeに「WebアプリのAPIサーバーを作って」と頼んだ。5分で動くコードが出てきた。テストも通った。「すごい、もうできた」と思った。

翌日、セキュリティに詳しい知人に見せた。開口一番、「これ、このまま公開したらまずいよ」と言われた。

何がまずかったのか。

## 「動く」と「安全」は別物

知人が指摘したのは3つだった。

**1つ目: 誰でもアクセスできる状態**

APIが「どこからのリクエストでも受け付ける」設定になっていた。自分のWebサイトからだけでなく、世界中の誰からでもAPIを叩ける。開発中は便利だが、本番でこの設定のままにすると、悪意のあるサイトから自分のAPIを勝手に使われる可能性がある。

技術的にはCORS（Cross-Origin Resource Sharing）の設定が`*`（全許可）になっていた。

**2つ目: セキュリティヘッダーがない**

ブラウザには「このサイトは安全ですよ」と伝えるための仕組みがある。HTTPレスポンスヘッダーというもので、たとえば「このサイトのコンテンツを他のサイトに埋め込むな」とか「JavaScriptの読み込み元をこのドメインだけに限定しろ」といった指示を出せる。

Claude Codeが作ったコードにはこのヘッダーが一切なかった。Express.jsなら`helmet()`という1行を追加するだけで基本的なセキュリティヘッダーが全部設定されるのだが、AIは指示されない限り追加しない。

**3つ目: ユーザー入力をそのまま使っていた**

「ユーザーが入力した名前で検索する」機能で、ユーザーの入力がそのままデータベースのクエリに組み込まれていた。

これが危険な理由は簡単だ。ユーザーが名前の代わりに「全データを消せ」という命令をデータベースに送り込めてしまう。「SQLインジェクション」と呼ばれる攻撃で、Webアプリの脆弱性ランキング（OWASP Top 10）で常にトップクラスに入る古典的な問題だ。

## なぜAIはセキュリティを省略するのか

AIは「動くコードを書く」ことに最適化されている。テストが通る、エラーが出ない、要求された機能が動く——これがAIの成功条件だ。

セキュリティ対策は「動くかどうか」には影響しない。helmet()がなくてもアプリは動く。CORSが全許可でもアプリは動く。SQLインジェクション対策がなくてもアプリは動く。

だからAIは省略する。悪意があるわけではない。「明示的に指示されていないことはやらない」というAIの特性の結果だ。

CLAUDE.mdに「セキュリティに注意して」と書いても、「注意する」の具体的な定義がないから効果は薄い。

## hookで自動チェックする

そこでhookを使う。hookはClaude Codeがコードを書いた直後に自動で内容をチェックする仕組みだ。

たとえば「SQLインジェクションの危険があるコードを検出する」hookを入れておくと、Claudeがデータベースのクエリを文字列結合で書いた瞬間に警告が出る。Claudeは指摘を受けて、安全な書き方（パラメータ化クエリ）に修正する。

```bash
# セキュリティチェック用hookをまとめてインストール
npx cc-safe-setup --shield
```

これで入るhookの中から、初心者が特に知っておくべきものを3つ紹介する。

### ユーザー入力をそのまま使うな（sql-injection-detect）

データベースへのクエリでユーザー入力を文字列結合していたら警告する。

```
⚠ 警告: ユーザー入力がSQLクエリに直接組み込まれています
→ パラメータ化クエリを使ってください
```

SQLインジェクションは「名前のところに命令文を入れる」攻撃。対策は「ユーザーの入力はあくまで"値"として扱い、"命令"として解釈させない」こと。hookがこのパターンを自動で見つけてくれる。

### セキュリティヘッダーを忘れるな（no-helmet-missing）

Express.jsのコードで`helmet()`が使われていなかったら警告する。

```
⚠ 警告: helmet()ミドルウェアが見つかりません
→ app.use(helmet()) を追加してセキュリティヘッダーを設定してください
```

helmet()は「Webサイトの玄関に鍵をかける」ようなもの。1行追加するだけで、よくある攻撃パターンをまとめて防げる。

### パスワードをハードコードするな（no-default-credentials）

`password: "admin"` のようなデフォルトパスワードがコードに書かれていたら警告する。

```
⚠ 警告: デフォルト認証情報がハードコードされています
→ 環境変数を使ってください
```

「とりあえずadmin/adminで動かす」つもりが、そのまま本番に出てしまう。これもよくある事故だ。

## まとめ: AIに任せていいこと、チェックすべきこと

AIにコードを書かせること自体は問題ない。5分で動くAPIサーバーが出てくるのは本当にすごい。

ただし「動く」と「安全」は別物だ。AIは動くコードを書くことには長けているが、セキュリティ対策は明示的に指示しないと省略する。

hookを入れておけば、AIが書いたコードを自動でチェックして「ここ危ないよ」と教えてくれる。自分がセキュリティの専門家でなくても、最低限の安全対策が入っているか確認できる。
