Claudeがコードを書く時、セキュリティの穴を作らないか？15のhookで自動チェック。

## インストール

```bash
npx cc-safe-setup --shield
```

個別に入れるなら:

```bash
npx cc-safe-setup --install-example <hook名>
```

## インジェクション系

### 1. sql-injection-detect
SQLの文字列結合を検出。パラメータ化クエリを促す。

### 2. no-exec-user-input
`exec(req.body...)`パターンを検出。コマンドインジェクション防止。

### 3. no-path-join-user-input
`path.join(userInput)`を検出。パストラバーサル防止。

### 4. no-xml-external-entity
XMLパーサーのXXEパターンを検出。

## XSS系

### 5. no-dangerouslySetInnerHTML
ReactのXSSリスクパターンを検出。

### 6. no-innerhtml
`innerHTML =`を検出。DOMベースXSS防止。

### 7. no-document-write
`document.write()`を検出。

## 認証・認可

### 8. no-default-credentials
`password: "admin"`のようなデフォルト認証情報を検出。

### 9. no-jwt-in-url
URLクエリパラメータ内のJWTトークンを検出。

### 10. no-cleartext-storage
localStorageへのパスワード/トークン保存を検出。

## 通信

### 11. check-tls-version
TLS 1.2未満の指定を検出。

### 12. no-wildcard-cors
`Access-Control-Allow-Origin: *`を検出。

### 13. check-https-redirect
HTTPリダイレクトでHTTPS未使用を検出。

## サーバー設定

### 14. no-helmet-missing
Express.jsでhelmet()未使用を検出。

### 15. check-cookie-flags
Cookie設定でsecureフラグ未使用を検出。

## まとめ

```bash
npx cc-safe-setup --shield
```
