Claude Codeはデフォルトで安全装置がない。`rm -rf /`も、`git push --force origin main`も、`.env`のコミットも、聞かれたら「はい」を押せば通る。Auto Modeなら聞かれもしない。

これは設計思想として正しい（ツールに制限をかけすぎると使えない）が、**自律運転するなら安全装置は自分で入れる必要がある**。

この記事では「何が危険で」「どう防いで」「どこから始めればいいか」を整理する。

## 実際に起きたインシデント

Claude CodeのGitHub Issuesから抜粋:

| インシデント | Issue |
|---|---|
| NTFS junction経由で`rm -rf`がユーザーディレクトリ全削除 | [#36339](https://github.com/anthropics/claude-code/issues/36339) |
| `Remove-Item -Recurse -Force`で未pushソース破壊 | [#37331](https://github.com/anthropics/claude-code/issues/37331) |
| Mac全ファイルシステム削除 | [#36233](https://github.com/anthropics/claude-code/issues/36233) |
| 48個のBashコマンドで認証情報を自動スキャン | [#37845](https://github.com/anthropics/claude-code/issues/37845) |

これらは全て、hookを1つ入れていれば防げた。

## hookとは

Claude Codeの`settings.json`に設定するスクリプト。ツール実行の**前**（PreToolUse）や**後**（PostToolUse）に自動で走り、危険な操作をブロックできる。

```json
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "command",
        "command": "bash ~/.claude/hooks/destructive-guard.sh"
      }]
    }]
  }
}
```

上の例では、Bashツールが呼ばれるたびに`destructive-guard.sh`が実行される。スクリプトが`exit 2`を返すと操作がブロックされる。

## 10秒で始める方法

```bash
npx cc-safe-setup
```

これだけで8つの安全hookがインストールされる:

1. **destructive-guard** — `rm -rf /`, `git reset --hard`, `chmod 777 /`をブロック
2. **branch-guard** — main/masterへの直接push、force-pushをブロック
3. **secret-guard** — `.env`, 秘密鍵の`git add`をブロック
4. **syntax-check** — ファイル編集後に構文チェック（壊れたまま次に進まない）
5. **context-monitor** — コンテキスト使用量を監視
6. **npm-publish** — 意図しない`npm publish`を防止
7. **response-budget** — 応答回数の上限管理
8. **commit-msg** — コミットメッセージの品質チェック

## もっと細かく設定したい場合

358個のexample hookから選んでインストールできる:

```bash
# 一覧を見る
npx cc-safe-setup --examples

# 特定のhookをインストール
npx cc-safe-setup --install-example credential-exfil-guard

# 自然言語で探す
npx cc-safe-setup --suggest
```

カテゴリの例:
- **セキュリティ**: credential漏洩防止、path traversal検知、deploy前のcommit強制
- **コード品質**: テスト比率チェック、TODO残しブロック、import整理
- **運用**: セッション時間制限、ファイル変更追跡、自動バックアップ

## 今の状態を診断する

```bash
npx cc-health-check
```

20項目をチェックして100点満点でスコアを出す。何が足りないか、どのhookを入れればいいかも教えてくれる。

## まとめ

| やりたいこと | コマンド |
|---|---|
| まず安全にする | `npx cc-safe-setup` |
| 今の状態を知る | `npx cc-health-check` |
| hookを探す | `npx cc-safe-setup --examples` |
| 特定hookを入れる | `npx cc-safe-setup --install-example <名前>` |

Claude Codeを自律運転させるなら、安全装置は最初に入れる。インシデントが起きてからでは遅い。
