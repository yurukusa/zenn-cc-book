2026年4月、Claude Codeのworktree関連の問題が急増している。
具体的に何が起きているか:
- **別セッションが使っているworktreeを勝手に削除される**（[#40850](https://github.com/anthropics/claude-code/issues/40850)）
- **git cleanがworktree境界を越えて波及する**
- **未マージのコミットがworktree除去時に消える**
- **スケジュールタスクが38個のPRを重複作成**（[#47029](https://github.com/anthropics/claude-code/issues/47029)）
Claude Codeのhook機能（`PreToolUse`）で、これらを**自動的に**防げる。
```bash
COMMAND=$(cat | jq -r '.tool_input.command // empty' 2>/dev/null)
[ -z "$COMMAND" ] && exit 0
if echo "$COMMAND" | grep -qE 'git\s+worktree\s+(remove|prune)'; then
    echo "BLOCKED: Cannot remove git worktrees — other sessions may depend on them." >&2
    echo "List worktrees first: git worktree list" >&2
    exit 2
fi
exit 0
```
`exit 2`がポイント。`exit 1`は「警告して続行」、`exit 2`は「ブロック（実行させない）」。
```bash
COMMAND=$(cat | jq -r '.tool_input.command // empty' 2>/dev/null)
[ -z "$COMMAND" ] && exit 0
echo "$COMMAND" | grep -qE '\bgit\s+(clean|reset|checkout\s+--|stash\s+drop)' || exit 0
GITDIR=$(git rev-parse --git-dir 2>/dev/null)
if echo "$GITDIR" | grep -q "worktrees"; then
    MAIN_DIR=$(git rev-parse --path-format=absolute --git-common-dir 2>/dev/null | sed 's|/.git$||')
    echo "WARNING: You are in a git worktree." >&2
    echo "Main working tree: $MAIN_DIR" >&2
    echo "Destructive git operations may affect the main tree." >&2
fi
exit 0
```
```bash
COMMAND=$(cat | jq -r '.tool_input.command // empty' 2>/dev/null)
[ -z "$COMMAND" ] && exit 0
echo "$COMMAND" | grep -qE '\bgh\s+pr\s+create\b' || exit 0
BRANCH=$(git rev-parse --abbrev-ref HEAD 2>/dev/null)
[ -z "$BRANCH" ] && exit 0
EXISTING=$(gh pr list --head "$BRANCH" --state open --json number --jq '.[0].number' 2>/dev/null)
if [ -n "$EXISTING" ]; then
    echo "BLOCKED: Open PR #$EXISTING already exists for '$BRANCH'." >&2
    exit 2
fi
exit 0
```
