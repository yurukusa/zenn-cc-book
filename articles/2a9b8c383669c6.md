Claude Codeにコードを書かせていた。Pythonファイルを1つ修正するだけの簡単なタスク。

修正が終わった。テストを走らせた。Syntax Error。

CCは「直します」と言って修正した。またSyntax Error。「直します」。Syntax Error。このループが10回続いた。

原因はシンプルだった。**編集した直後に構文チェックが走っていなかった**。壊れたコードの上にさらに修正を重ねるから、どんどん壊れていく。

> **Note:** この記事は2026年3月時点の情報に基づいている。

## PostToolUseに構文チェックを仕込む

Claude Code Hooksには「ツール実行後に自動でスクリプトを走らせる」仕組みがある。PostToolUse。

EditやWriteの後に構文チェックを走らせれば、壊れた瞬間に気づける。

```json
// ~/.claude/settings.json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "bash ~/.claude/hooks/syntax-check.sh"
          }
        ]
      }
    ]
  }
}
```

syntax-check.shは、ファイルの拡張子を見て適切なチェッカーを走らせるだけ。

- Python: `python3 -m py_compile`
- Shell: `bash -n`
- JSON: `python3 -m json.tool`
- JavaScript/TypeScript: `node --check`

**CC補足: PostToolUseの仕組み**
PostToolUseフックはCCがツールを実行した直後にトリガーされる。`matcher`でツール名をフィルタできる。フックの終了コード0以外でブロックメッセージをCCに返せるため、「構文エラーがあるから次に進むな」と強制できる。

これを入れた瞬間、構文エラーループがほぼゼロになった。壊した直後に「壊れてるぞ」と返ってくるから、CCは即座に正しい修正ができる。

## もう1つの穴: bashの出力エラーを見落とす

構文チェックとは別の問題もあった。

`npm install` がエラーで失敗しているのに、CCはそのまま次の作業に進んだ。エラーメッセージが出ているのに見落としていた。

bash実行の出力をJSONLログに記録するhookを追加した。エラーパターンを検知して、次の外部アクション（git pushなど）をブロックする。

```json
{"ts":"2026-02-28T12:00:00Z","tool":"Bash","file":"npm install","exit_code":1,"error_pattern":"ENOENT"}
```

「エラーを記録する仕組み」と「エラーがあるときにブロックする仕組み」がセットで初めて機能する。片方だけでは意味がない。

## 「完了」の定義がなかった

CCが「記事を投稿しました！」と報告した。確認したら誤字がある。タグが間違っている。URLが死んでいる。

CCにとっての「完了」は「投稿ボタンを押した瞬間」だった。

完了の定義（DoD）チェックリストを作った。

```markdown
## 記事投稿のDoD
- [ ] 投稿後にURLが開けることを確認した
- [ ] タイトルとタグが正しいことを確認した
- [ ] スクリーンショットで公開状態を目視確認した
```

CLAUDE.mdに「DoDを満たすまで完了と報告するな」と書いた。以来、報告と実態のズレが消えた。

## 自分のアウトプットを検証させる

「GitHubにpushしました」。確認したら失敗していた。

CLAUDE.mdに検証ステップを明記した。

```markdown
## 検証ルール
- git push後: git log --oneline -1 origin/main で確認
- 記事投稿後: URLにアクセスして公開状態を目視確認
- ファイル作成後: ls -la で存在確認
```

やったと言うなら、やれたことを証明しろ。

## 4つ入れた結果

| チェック | 解決する問題 |
|---------|------------|
| 構文チェックhook | 編集直後のエラーを即検知 |
| エラー検知hook | bashエラーの見落とし防止 |
| DoDチェックリスト | 「完了」の定義ズレ |
| 検証ステップ | 「やった」と「やれた」のギャップ |

全部入れてから気づいたのは、CCの問題じゃないということ。仕組みがないのが問題だった。仕組みを入れれば、CCはちゃんと動く。
